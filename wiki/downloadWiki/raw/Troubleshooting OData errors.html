This page provides resources on how to troubleshoot errors you may find while using OData through datajs.

Typically the first thing to do is to get clear error messages. If the server is running WCF Data Services, you can turn on exception details as described in Debugging WCF Data Services.

Other tools to look at what's going on include the following.

  • Fiddler. An easy to use visual tool for inspecting, generating and replaying HTTP requests & responses for debugging purposes.
  • Network Tab in IE9. An easy-to-use capturing tool to see what's going on.
  • Firebug. The classic set of debugging tools for Firefox.
